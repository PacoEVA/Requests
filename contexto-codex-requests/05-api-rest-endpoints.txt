API REST RECOMENDADA

BASE URL
/api

AUTENTICACIÓN

POST /api/auth/login
Body:
{
  "username": "admin",
  "password": "123456"
}
Respuesta:
{
  "token": "jwt",
  "user": { "id": 1, "username": "admin", "fullName": "Admin", "role": "Admin", "requirePasswordChange": false }
}

POST /api/auth/logout
- Puede ser client-side si se usa JWT Bearer.

GET /api/auth/me
- Requiere token.
- Retorna usuario actual.

POST /api/auth/change-password
- Requiere token.
Body:
{
  "currentPassword": "old",
  "newPassword": "new"
}

POST /api/auth/force-change-password
- Requiere token.
- Usado cuando RequirePasswordChange = true.
Body:
{
  "newPassword": "new"
}

EMPLEADOS SIN LOGIN

POST /api/employees/identify
Body:
{
  "name": "Juan Pérez",
  "departmentId": 1,
  "employeeCode": "EMP001",
  "phoneOrExtension": "250"
}
Respuesta:
{
  "employee": { ... },
  "publicToken": "uuid"
}

GET /api/employees/me
Headers:
- x-employee-token: uuid
Respuesta:
{
  "employee": { ... }
}

PUT /api/employees/me
Headers:
- x-employee-token: uuid
Body:
{
  "name": "Juan Pérez",
  "departmentId": 1,
  "phoneOrExtension": "250"
}

REQUISICIONES DEL EMPLEADO

POST /api/requisitions
Headers:
- x-employee-token: uuid
Body:
{
  "priority": "Media",
  "generalComment": "Necesito estos materiales para el área de contabilidad",
  "items": [
    {
      "materialId": 1,
      "quantityRequested": 5,
      "unitOfMeasure": "Resma",
      "comment": "Papel tamaño carta"
    },
    {
      "manualMaterialName": "Marcadores rojos",
      "quantityRequested": 3,
      "unitOfMeasure": "Unidad",
      "comment": "No aparece en catálogo"
    }
  ]
}

GET /api/requisitions/my
Headers:
- x-employee-token: uuid
Query:
- status
- dateFrom
- dateTo
- search
- page
- pageSize

GET /api/requisitions/my/:id
Headers:
- x-employee-token: uuid

PATCH /api/requisitions/my/:id/cancel
Headers:
- x-employee-token: uuid
Body:
{
  "reason": "Ya no necesito el material"
}

COMENTARIOS

GET /api/requisitions/:id/comments
- Para empleado: requiere x-employee-token y validar que sea dueño.
- Para admin: requiere JWT.

POST /api/requisitions/:id/comments
Body:
{
  "message": "Comentario de prueba"
}
