STACK DEL PROYECTO

Frontend:
- React.
- TypeScript.
- React Router.
- Context API o Zustand para estado global sencillo.
- Socket.IO Client para comunicación en tiempo real.
- Librería de formularios opcional: React Hook Form.
- Validación opcional: Zod o Yup.
- UI: puede usarse CSS normal, TailwindCSS o una librería de componentes si se decide luego.

Backend:
- Node.js.
- ExpressJS.
- TypeScript preferiblemente.
- Socket.IO Server.
- SQL Server usando el paquete mssql.
- bcrypt para contraseñas.
- JWT para autenticación del dashboard.
- dotenv para variables de entorno.
- cors configurado correctamente.

Base de datos:
- SQL Server.
- Tablas relacionales.
- Uso de transacciones para crear requisiciones con sus items.
- Stored procedures opcionales, pero no obligatorias.

Principios de arquitectura:
- Separar lógica de UI y lógica de API.
- No hacer consultas SQL directamente en controladores.
- Usar services para reglas de negocio.
- Usar repositories para acceso a SQL Server.
- Usar middleware de errores global.
- Usar validaciones en frontend y backend.
- El backend debe ser la fuente de verdad.
- Socket.IO se usa para notificar cambios, no como sustituto completo de la API REST.
- Temporalmente no implementar integración SAP. El inventario funciona solo contra SQL Server local de Requests.
