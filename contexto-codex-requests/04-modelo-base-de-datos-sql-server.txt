MODELO DE BASE DE DATOS SQL SERVER

NOTAS GENERALES
- Usar IDs tipo INT IDENTITY o UNIQUEIDENTIFIER. Para simplicidad inicial se recomienda INT IDENTITY.
- Usar CreatedAt, UpdatedAt y IsActive donde aplique.
- No crear tabla Attachments.
- No crear tablas de sucursales.
- No manejar inventario, stocks ni movimientos de inventario.
- Los materiales no llevan categoria ni unidad de medida.

1. Departments
Campos:
- Id INT IDENTITY PRIMARY KEY
- Name NVARCHAR(150) NOT NULL
- Description NVARCHAR(300) NULL
- IsActive BIT NOT NULL DEFAULT 1
- CreatedAt DATETIME2 NOT NULL DEFAULT SYSUTCDATETIME()
- UpdatedAt DATETIME2 NULL

2. Employees
Campos:
- Id INT IDENTITY PRIMARY KEY
- PublicToken UNIQUEIDENTIFIER NOT NULL DEFAULT NEWID()
- Name NVARCHAR(150) NOT NULL
- DepartmentId INT NOT NULL FOREIGN KEY REFERENCES Departments(Id)
- EmployeeCode NVARCHAR(50) NULL
- PhoneOrExtension NVARCHAR(50) NULL
- IsActive BIT NOT NULL DEFAULT 1
- CreatedAt DATETIME2 NOT NULL DEFAULT SYSUTCDATETIME()
- UpdatedAt DATETIME2 NULL

Reglas:
- PublicToken se usa para identificar al empleado desde el navegador.
- EmployeeCode puede ser único si la empresa lo requiere.

3. Roles
Campos:
- Id INT IDENTITY PRIMARY KEY
- Name NVARCHAR(50) NOT NULL UNIQUE

Seed obligatorio:
- Admin
- Compras
- Supervisor

No crear rol Consulta.

4. InternalUsers
Campos:
- Id INT IDENTITY PRIMARY KEY
- Username NVARCHAR(80) NOT NULL UNIQUE
- FullName NVARCHAR(150) NOT NULL
- PasswordHash NVARCHAR(255) NOT NULL
- RoleId INT NOT NULL FOREIGN KEY REFERENCES Roles(Id)
- DepartmentId INT NULL FOREIGN KEY REFERENCES Departments(Id)
- IsActive BIT NOT NULL DEFAULT 1
- RequirePasswordChange BIT NOT NULL DEFAULT 0
- LastLoginAt DATETIME2 NULL
- CreatedAt DATETIME2 NOT NULL DEFAULT SYSUTCDATETIME()
- UpdatedAt DATETIME2 NULL

5. Materials
Campos:
- Id INT IDENTITY PRIMARY KEY
- ItemCode NVARCHAR(80) NULL UNIQUE FILTRADO
- Name NVARCHAR(200) NOT NULL
- Description NVARCHAR(500) NULL
- IsActive BIT NOT NULL DEFAULT 1
- IsRequestable BIT NOT NULL DEFAULT 1
- CreatedAt DATETIME2 NOT NULL DEFAULT SYSUTCDATETIME()
- UpdatedAt DATETIME2 NULL

Reglas:
- Los materiales se administran localmente en Requests.
- ItemCode permite relacionar el material con un codigo interno o externo.

6. RequisitionStatuses
Campos:
- Id INT IDENTITY PRIMARY KEY
- Code NVARCHAR(50) NOT NULL UNIQUE
- Name NVARCHAR(80) NOT NULL
- SortOrder INT NOT NULL
- IsFinal BIT NOT NULL DEFAULT 0

Seed recomendado:
- PENDING / Pendiente
- IN_REVIEW / En revisión
- APPROVED / Aprobada
- REJECTED / Rechazada
- IN_PURCHASE / En compra
- READY_TO_DELIVER / Lista para entregar
- PARTIALLY_DELIVERED / Entrega parcial
- DELIVERED / Entregada
- CANCELLED / Cancelada

Aunque se eliminó configuración dinámica de estados, esta tabla puede existir como catálogo fijo/seedeado.

7. Requisitions
Campos:
- Id INT IDENTITY PRIMARY KEY
- Code NVARCHAR(30) NOT NULL UNIQUE
- EmployeeId INT NOT NULL FOREIGN KEY REFERENCES Employees(Id)
- StatusId INT NOT NULL FOREIGN KEY REFERENCES RequisitionStatuses(Id)
- Priority NVARCHAR(20) NOT NULL DEFAULT 'Media' -- Baja, Media, Alta, Urgente
- GeneralComment NVARCHAR(1000) NULL
- AssignedToUserId INT NULL FOREIGN KEY REFERENCES InternalUsers(Id)
- CreatedAt DATETIME2 NOT NULL DEFAULT SYSUTCDATETIME()
- UpdatedAt DATETIME2 NULL
- ClosedAt DATETIME2 NULL
- CancelledAt DATETIME2 NULL
- CancelReason NVARCHAR(1000) NULL
- RejectedAt DATETIME2 NULL
- RejectReason NVARCHAR(1000) NULL

8. RequisitionItems
Campos:
- Id INT IDENTITY PRIMARY KEY
- RequisitionId INT NOT NULL FOREIGN KEY REFERENCES Requisitions(Id)
- MaterialId INT NULL FOREIGN KEY REFERENCES Materials(Id)
- ManualMaterialName NVARCHAR(200) NULL
- QuantityRequested DECIMAL(18,4) NOT NULL
- QuantityApproved DECIMAL(18,4) NULL
- QuantityDelivered DECIMAL(18,4) NULL DEFAULT 0
- Comment NVARCHAR(500) NULL
- CreatedAt DATETIME2 NOT NULL DEFAULT SYSUTCDATETIME()

Reglas:
- MaterialId puede ser NULL si el usuario escribió material manual.
- Si MaterialId es NULL, ManualMaterialName debe tener valor.
- QuantityRequested debe ser > 0.

9. RequisitionComments
Campos:
- Id INT IDENTITY PRIMARY KEY
- RequisitionId INT NOT NULL FOREIGN KEY REFERENCES Requisitions(Id)
- AuthorType NVARCHAR(20) NOT NULL -- EMPLOYEE, INTERNAL_USER
- EmployeeId INT NULL FOREIGN KEY REFERENCES Employees(Id)
- InternalUserId INT NULL FOREIGN KEY REFERENCES InternalUsers(Id)
- Message NVARCHAR(2000) NOT NULL
- CreatedAt DATETIME2 NOT NULL DEFAULT SYSUTCDATETIME()

No incluir archivos adjuntos.

10. RequisitionHistory
Campos:
- Id INT IDENTITY PRIMARY KEY
- RequisitionId INT NOT NULL FOREIGN KEY REFERENCES Requisitions(Id)
- Action NVARCHAR(80) NOT NULL -- CREATED, STATUS_CHANGED, APPROVED, REJECTED, CANCELLED, DELIVERED, COMMENT_ADDED, ASSIGNED
- PreviousStatusId INT NULL FOREIGN KEY REFERENCES RequisitionStatuses(Id)
- NewStatusId INT NULL FOREIGN KEY REFERENCES RequisitionStatuses(Id)
- PerformedByType NVARCHAR(20) NOT NULL -- EMPLOYEE, INTERNAL_USER, SYSTEM
- EmployeeId INT NULL FOREIGN KEY REFERENCES Employees(Id)
- InternalUserId INT NULL FOREIGN KEY REFERENCES InternalUsers(Id)
- Notes NVARCHAR(1000) NULL
- CreatedAt DATETIME2 NOT NULL DEFAULT SYSUTCDATETIME()

11. Notifications
Campos:
- Id INT IDENTITY PRIMARY KEY
- RecipientType NVARCHAR(20) NOT NULL -- EMPLOYEE, INTERNAL_USER, ROLE
- EmployeeId INT NULL FOREIGN KEY REFERENCES Employees(Id)
- InternalUserId INT NULL FOREIGN KEY REFERENCES InternalUsers(Id)
- RoleId INT NULL FOREIGN KEY REFERENCES Roles(Id)
- RequisitionId INT NULL FOREIGN KEY REFERENCES Requisitions(Id)
- Title NVARCHAR(150) NOT NULL
- Message NVARCHAR(500) NOT NULL
- Type NVARCHAR(50) NOT NULL -- REQUISITION_CREATED, STATUS_CHANGED, COMMENT_ADDED, ETC
- IsRead BIT NOT NULL DEFAULT 0
- CreatedAt DATETIME2 NOT NULL DEFAULT SYSUTCDATETIME()
- ReadAt DATETIME2 NULL

TABLAS ELIMINADAS / NO CREAR

- Attachments.
- RequisitionAttachments.
- CommentAttachments.
- SapSyncLog.
- MaterialCategories.
- Inventory.
- InventoryMovements.
