PROYECTO: Requests - Plataforma interna de requisiciones de material gastable

FUENTE BASE DEL PROYECTO
El sistema será una plataforma web donde los empleados de una empresa puedan realizar requisiciones de material gastable al departamento de Compras. La respuesta de la requisición debe comunicarse en tiempo real dentro de la misma plataforma.

LADO DEL EMPLEADO
- No debe manejar login tradicional.
- Debe existir una forma de identificar a cada empleado de forma independiente.
- Al cerrar la aplicación web, la información del empleado debe persistir.
- El empleado debe poder volver luego y crear otra requisición sin problemas.

LADO DE COMPRAS / ADMINISTRACIÓN
- Debe existir un dashboard privado.
- Esta parte sí debe tener login.
- El encargado debe poder ver y gestionar requisiciones.
- Deben mostrarse métricas como pendientes, completadas, entregadas y canceladas.

STACK DEFINIDO
- Frontend: React + TypeScript.
- Backend: ExpressJS.
- Tiempo real: Socket.IO.
- Base de datos: SQL Server.

DECISIONES VALIDADAS POR EL USUARIO
1. Eliminar módulos de sucursales:
   - SUC-01 Crear sucursales.
   - SUC-02 Editar sucursales.
   - SUC-03 Filtrar requisiciones por sucursal.

2. Eliminar configuraciones generales:
   - CONF-01 Configurar nombre de empresa.
   - CONF-02 Configurar estados permitidos.
   - CONF-04 Configurar tiempo máximo pendiente.

3. Actualizar AUTH-06:
   - Deben existir roles, pero se elimina el rol de consulta.
   - Roles permitidos inicialmente: Admin, Compras, Supervisor.

4. Actualizar AUTH-08:
   - La recuperación/restablecimiento de contraseña debe ser sencilla.
   - Debe poder realizarse dentro de la misma plataforma.
   - No depender inicialmente de correo electrónico.
   - Recomendación: el Admin puede resetear la contraseña de un usuario interno desde el módulo de usuarios.

5. Actualizar DASH-12:
   - Debe existir una parte de estadísticas.
   - Debe mostrar departamento con más solicitudes, material más utilizado, solicitudes por estado, tendencias y otros indicadores útiles.

6. Remover archivos adjuntos completamente:
   - No implementar subida de archivos en requisiciones.
   - No implementar archivos en comentarios.
   - No crear tabla Attachments.
   - No crear endpoints de upload.
   - No usar multer para esta primera especificación.

7. Inventario local:
   - Temporalmente se removió la integración con SAP Business One.
   - El sistema debe funcionar solamente con la base de datos local de Requests.
   - No crear endpoints, módulos, tablas ni variables de entorno relacionadas con SAP en esta etapa.

OBJETIVO DE ESTOS ARCHIVOS
Estos archivos sirven como contexto para Codex. Deben usarse para construir el proyecto desde cero con una arquitectura limpia, modular y mantenible.
