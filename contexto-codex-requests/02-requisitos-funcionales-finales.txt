REQUISITOS FUNCIONALES FINALES DEL SISTEMA

1. MÓDULO DEL EMPLEADO SIN LOGIN

EMP-01 Registro inicial sin login
- El empleado debe identificarse inicialmente con datos básicos.
- Campos sugeridos: nombre, departamento, teléfono/extensión y código de empleado si existe.
- No se usa usuario y contraseña para empleados solicitantes.

EMP-02 Guardar identidad en navegador
- Guardar un employeeId o employeeToken en localStorage.
- Al volver a abrir la aplicación, el sistema debe reconocer al empleado.

EMP-03 Crear empleado automáticamente en SQL Server
- Aunque el empleado no tenga login, debe existir un registro en la base de datos.
- Cada requisición debe quedar asociada a un EmployeeId.

EMP-04 Editar datos del empleado
- El empleado puede actualizar su nombre, departamento o teléfono.

EMP-05 Validar empleado por código interno
- Si la empresa usa códigos de empleado, se puede evitar duplicidad usando ese código.

EMP-06 Restablecer identidad del navegador
- Permitir cambiar de empleado si varias personas usan el mismo equipo.

EMP-07 Ver perfil básico del empleado
- Mostrar nombre y departamento activo en el área del empleado.

2. MÓDULO DE CREACIÓN DE REQUISICIONES

REQ-01 Crear requisición
- El empleado puede crear una solicitud de material gastable.

REQ-02 Generar código automático
- Código recomendado: REQ-YYYY-000001.
- Ejemplo: REQ-2026-000001.

REQ-03 Agregar varios materiales en una requisición
- Una requisición puede contener múltiples líneas de materiales.

REQ-04 Seleccionar material desde catálogo
- El empleado puede seleccionar materiales registrados.

REQ-05 Escribir material manualmente
- Permitir opción de material no catalogado como texto libre.
- Debe marcarse como material manual/no catalogado.

REQ-06 Indicar cantidad solicitada
- Cada línea debe tener cantidad mayor que 0.

REQ-07 Indicar unidad de medida
- Ejemplos: unidad, caja, paquete, resma, galón.

REQ-08 Agregar comentario general
- Campo para explicar motivo o necesidad de la requisición.

REQ-09 Agregar comentario por material
- Cada línea de material puede tener una observación.

REQ-10 Seleccionar prioridad
- Valores: Baja, Media, Alta, Urgente.

REQ-12 Confirmación antes de enviar
- Mostrar resumen antes de crear la requisición.

REQ-13 Validación de campos
- No permitir requisiciones vacías.
- No permitir cantidades inválidas.
- No permitir empleado no identificado.

REQ-14 Cancelar requisición propia
- El empleado puede cancelar una solicitud si todavía no fue aprobada, entregada o cerrada.

IMPORTANTE: Se removieron archivos adjuntos. No implementar REQ-11.

5. LOGIN PARA COMPRAS / ADMIN / SUPERVISOR

AUTH-01 Login de usuario interno
- Usuarios internos acceden con usuario y contraseña.

AUTH-02 Contraseña encriptada
- Usar bcrypt.

AUTH-03 JWT o sesión segura
- Recomendado: JWT guardado en memoria/localStorage o cookie httpOnly.
- Para simplicidad inicial, JWT Bearer es aceptable.

AUTH-04 Logout
- Cerrar sesión del dashboard.

AUTH-05 Proteger rutas del dashboard
- Rutas privadas en React y middleware auth en Express.

AUTH-06 Roles de usuario
- Roles permitidos:
  - Admin.
  - Compras.
  - Supervisor.
- Rol Consulta eliminado.

AUTH-07 Cambiar contraseña
- Usuario interno puede cambiar su contraseña si conoce la anterior.

AUTH-08 Recuperación/restablecimiento sencillo en plataforma
- Debe realizarse dentro del sistema.
- No requiere correo en primera versión.
- Flujo recomendado:
  1. Un Admin entra al módulo Usuarios.
  2. Selecciona el usuario interno.
  3. Presiona "Restablecer contraseña".
  4. El sistema genera una contraseña temporal o permite escribir una nueva.
  5. Se marca RequirePasswordChange = 1.
  6. En el siguiente login, el usuario debe cambiar la contraseña.

AUTH-09 Usuario activo/inactivo
- Un usuario interno puede bloquearse sin eliminar historial.
