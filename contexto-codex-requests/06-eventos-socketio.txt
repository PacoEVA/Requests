EVENTOS SOCKET.IO

OBJETIVO
Socket.IO se utilizará para notificaciones y actualizaciones en tiempo real entre:
- Empleado solicitante.
- Dashboard de Compras/Admin/Supervisor.

CONEXIÓN

Al conectar:
- Si es empleado, debe enviar employeeToken o employeeId.
- Si es admin, debe enviar JWT o autenticarse después de conectar.

Rooms recomendadas:
- employee:{employeeId}
- role:Admin
- role:Compras
- role:Supervisor
- requisition:{requisitionId}
- dashboard:admins

EVENTOS DEL CLIENTE AL SERVIDOR

employee:join
Payload:
{
  "employeeToken": "uuid"
}
Acción:
- Validar token.
- Asociar socket a room employee:{employeeId}.

admin:join
Payload:
{
  "token": "jwt"
}
Acción:
- Validar JWT.
- Asociar socket a rooms del usuario.
- Si rol Admin o Compras, unir a dashboard:admins.

requisition:join
Payload:
{
  "requisitionId": 1
}
Acción:
- Validar permiso.
- Unir al room requisition:{id}.

requisition:leave
Payload:
{
  "requisitionId": 1
}

notification:read
Payload:
{
  "notificationId": 1
}

REGLAS IMPORTANTES
- No confiar en IDs recibidos por Socket.IO sin validar permisos.
- Las operaciones críticas deben hacerse por API REST.
- Socket.IO solo notifica resultados ya guardados.
- Siempre guardar primero en SQL Server y luego emitir evento.
- Si falla Socket.IO, la API REST debe seguir funcionando.
